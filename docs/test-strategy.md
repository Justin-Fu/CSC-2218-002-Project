1. System Overview
    The purpose of this system is to provide a study planner website to users. This project is targeted primarily to students, and will help them input and organize to-do assignments and deadlines. Some major features that is planned is that the user will be able to input and save data.


2. Testing Approach
    Due to the simplicity of this project, the best testing approach is to test the user interface and how users would interact with the website. The website would need to be as easy to navigate as well as simple. There will be some gaps in the testing coverage, since this is a basic project, one example of testing that would not be included is security vulnerability testing, due to the lack of resources and time. In addition, due to the limited scope of this project, there will be no actual testing when creating this project.

3. Test Environment
    The system would be tested on a web browser, and would simulate typical user interactions. This project needs to be tested on multiple different common web browsers, in order to ensure that it will be accessible to users.
